# @linkshield/extension-core

Single source of truth for LinkShield browser extension code. Chrome, Firefox, and Safari all sync from here.

## Why this exists

Before: same popup.html, block-page.js, welcome.html, content scripts — manually copied into three directories. Every popup change required three copies + manual testing that we didn't forget one. In practice: firefox/safari got stale.

**Now:** edit `packages/extension-core/src/...` once, run `scripts/build-extensions.sh`, all three browser builds refresh.

## What lives here

```

├── popup/              ← Main action popup (+ welcome.html, block-preview.html)
│   ├── popup.html
│   ├── popup.css
│   ├── popup.js
│   └── welcome.html
├── background/         ← Service worker / background script
│   └── index.js
├── content/            ← Content scripts injected into every page
│   ├── index.js           (main orchestrator)
│   ├── block-page.js      (STOP overlay for scam sites)
│   ├── privacy-audit.js
│   ├── security-score.js
│   ├── weekly-report.js
│   ├── breach-check.js
│   ├── tracking-cleaner.js
│   └── crypto-blocker.js
├── options/            ← Settings page
│   ├── options.html
│   └── options.js
└── utils/              ← Shared helpers
    ├── api.js             (API_BASE resolution + chrome.storage.local override)
    ├── bloom.js
    ├── local-scorer.js
    └── storage.js

public/icons/            ← Extension icons (16/32/48/128 px)
styles/badges.css        ← Link-badge CSS injected via content_scripts
```

## What DOESN'T live here

Each browser extension dir keeps its own:
- `manifest.json` — different format per browser (MV3 chrome, MV2 firefox, MV3 safari)
- `_locales/` — regenerated by `scripts/build-i18n.py` from `packages/i18n-strings/`
- `overrides/` (optional) — per-flavor file substitutions, see below

## Build

```bash
bash scripts/build-extensions.sh
```

This:
1. Clears `src/`, `public/`, `styles/` in each extension dir
2. Copies from `packages/extension-core/`
3. Applies `extension-{flavor}/overrides/` (if present) on top
4. For Firefox MV2: auto-injects a `chrome → browser` promise shim at the top of `background/index.js` (Firefox's `chrome.*` uses callbacks, our code uses `.then()`)
5. Re-runs `scripts/build-i18n.py` so locales stay in sync

Typical iteration:
```bash
# Edit packages/extension-core/src/popup/popup.html
bash scripts/build-extensions.sh
# Reload unpacked extension in Chrome → see changes
```

## Per-flavor overrides

If a specific browser needs a file that differs from the shared version, put it under that extension's `overrides/` directory with the same relative path. `build-extensions.sh` copies overrides AFTER the core sync, so they always win.

Example: Safari needs a different popup stylesheet
```
extension-safari/overrides/src/popup/popup.css  ← this wins over core's popup.css
```

**Rule of thumb:** if you find yourself reaching for overrides, think first if the divergence should actually live in core with a runtime flavor detection (e.g. `typeof browser !== "undefined"`).

## i18n integration

Code in this package uses `chrome.i18n.getMessage("key_name")`. The keys come from the final `_locales/<lang>/messages.json` produced by `scripts/build-i18n.py` from `packages/i18n-strings/`.

**Never** add literal English strings. Always use `chrome.i18n.getMessage` + add the key to `packages/i18n-strings/src/en.json` (and the 9 other locale files).

## 1. BACKEND API (FastAPI, Railway → api.cleanway.ai)

### Готово (где смотреть)
- 53 endpoints, 650 тестов, v0.2.0 в проде, 14/14 circuit breakers closed
- Stripe: idempotency, period dates, audit-log на каждое событие, business plan mapping, UNIQUE(user_id), 7 webhook handlers
- GDPR Art. 15 (export) + Art. 17 (30-day grace + hard purge cron) — `api/routers/user.py`, `api/services/account_purge.py`
- Soft-delete 410 lock — `api/services/auth.py` (Redis `deleted:{user_id}` TTL 30d)
- Audit-log — `api/services/audit_log.py` + 15 точек записи в `api/routers/`
- Sentry user context на каждом авторизованном запросе

- Нет hook'а для Sentry на лендинге → бэке (есть конфиг, но не вижу пробросов)
- Привязка к `addin.cleanway.ai` (DNS) — отдельный subdomain для Outlook plugin manifest, есть runbook но не настроен

### Тест на проде

## 2. LANDING (Next.js 15, cleanway.ai на Vercel)

### Готово (где смотреть)
- App Router + SSR, 10 локалей (en/ru/es/de/fr/it/pt/ar/hi/id) + RTL для арабского
- Supabase Auth magic-link: `landing/lib/supabase/{client,server}.ts`, callback в `landing/app/auth/callback/route.ts`
- Pricing 4 тира × PPP — `landing/app/[locale]/pricing/PricingClient.tsx`
- Stripe Checkout — клиент шлёт `POST /api/v1/payments/checkout`, получает `checkout_url`, редиректит
- Маршруты: `/`, `/[locale]/{signup,pricing,success,check/[domain],audit/[domain],business,family/join,privacy-policy,terms,ref/[code]}`
- Sentry wired (server/client/edge configs), SEO (sitemap 1000+ URLs, JSON-LD на pricing и check)
- E2E Playwright: `landing/e2e/{home,mobile}.spec.ts`

### Что осталось (важное)
- **Нет `/account` / `/settings` / `/dashboard`** — пользователь не может управлять своим аккаунтом с лендинга
- **Нет sign-out кнопки** — после логина пользователь "застрял" в авторизованном состоянии
- **Нет `/account/restore`** — после soft-delete письма с restore-ссылкой нужна landing-страница
- **Нет ссылки на Stripe Customer Portal** — бэк готов (`POST /payments/portal`), но landing не выставляет кнопку "Manage subscription"
- **Нет UX для 401/403/410** — все летят в generic alert или редиректят на /signup в петлю
- **90% страниц захардкожены на английском** — Pricing/Success/Business/Check/Signup-form НЕ переведены (только homepage)
- **OG-image для check/[domain]** — упоминается, но нужна верификация что генерится

### Тест в браузере (можно прямо сейчас)
- [ ] `/en`, `/ru`, `/ar` — проверить RTL и переводы
- [ ] `/pricing` → подсветка тира по гео (cookies/IP)
- [ ] `/signup` → magic link → callback → cookie ставится
- [ ] `/check/google.com` → SSR результат + JSON-LD

## 3. РАСШИРЕНИЯ БРАУЗЕРОВ (Chrome MV3 / Firefox MV2 / Safari MV3)

Архитектура: один источник `packages/extension-core/src/`, build-script собирает 3 таргета.

### Готово (где смотреть)
- 9 контент-скриптов: `index.js` (link badges, batch checking), `block-page.js` (full-screen STOP overlay + countdown), `privacy-audit.js`, `security-score.js`, `weekly-report.js`, `breach-check.js`, `tracking-cleaner.js`, `crypto-blocker.js`, `webmail.js`
- Background SW: LRU cache 1000 entries, 3s timeout, локальное скоринг + API fallback
- Popup: 4 состояния (scanning/safe/warning/danger), stats, recent threats, action tiles
- Skill levels (Kids/Regular/Granny/Pro) с PIN + voice alerts + device overrides
- Family Hub auto-fanout: `family-crypto.js`, `family-notifier.js`, `family-invite-url.js`
- i18n: 10 локалей через `_locales/{locale}/messages.json`
- 50-threat freemium soft-limit — celebratory nudge в popup
- Pricing v2 nudge UI: реализован в popup.html/.js (memory говорил pending — на самом деле сделано)
- Skill selector UI: реализован в options (memory говорил pending — на самом деле сделано)

### Что осталось
- **Кнопка "Report phishing"** в popup — UI есть (`popup.js:369` шлёт `SHOW_REPORT_DIALOG`), но **никто не слушает** это сообщение → silently ignored
- **Кнопка "Always trust this site"** — UI есть, но в обработчике стоит **wrong message type** (тоже `SHOW_REPORT_DIALOG` вместо записи в whitelist)
- **410 не обрабатывается** — на soft-deleted аккаунте уйдёт в generic ошибку
- **`_debugMode = true`** в `extension-core/src/content/index.js:12` — продакшен должен быть `false`
- **ZIP-ы устарели** — `cleanway-extension.zip` от 4 мая, нужен ре-билд через `scripts/build-extensions.sh`
- **Safari** — нет `.xcodeproj`, нужна обёртка через "Convert Web Extension" в Xcode
- **Firefox MV2** — asymmetry: 3 скрипта (`tracking-cleaner`, `crypto-blocker`, `breach-check`) работают только на webmail, не глобально. Решение либо принять либо мигрировать на MV3 (Firefox теперь поддерживает MV3)
- **0 автотестов** — только manual QA

## 4. MOBILE — iOS + Android (Expo + native VPN)

### Готово (где смотреть)
- Expo 52 + Expo Router v4, RN 0.76.9, TypeScript strict, ~2700 LOC screens + ~1300 LOC services
- Bundle ID `ai.cleanway.app` (iOS + Android)
- Auth (`app/(auth)/...`): Supabase email+password (signup, signin, reset) + SecureStore
- Screens: `index.tsx`, `history.tsx`, `score.tsx`, `settings.tsx`, `check.tsx`, `scanner.tsx`, `family.tsx`, `breach.tsx`, `report.tsx`, `onboarding.tsx`, `upgrade.tsx`
- Native VPN:
  - iOS: `mobile/native/ios/PacketTunnelProvider.swift` (268 LOC) — DNS intercept, NXDOMAIN на блок
  - Android: `mobile/native/android/CleanwayVpnService.kt` (301 LOC) — local VPN, DNS forward to 1.1.1.1
  - Оба покрыты unit-тестами: `DNSParserTests.swift` (13 тестов) + `DnsUtilTest.kt`
- Family Hub Phase 1+2: REST client + tweetnacl crypto в `mobile/src/lib/family-{api,crypto}.ts`
- Breach check (SHA-1 k-anonymity, первые 5 hex идут на сервер)
- Weekly report (on-device SQLite + percentile)
- 10 локалей через i18n

### Что осталось (приоритет)
- **🛑 Нет `eas.json`** — без него нельзя собрать ни TestFlight, ни Play Store. Нужно создать профили `preview` + `production`, прописать env vars (`EXPO_PUBLIC_API_URL`, `EXPO_PUBLIC_SUPABASE_URL`, `EXPO_PUBLIC_SUPABASE_ANON_KEY`)
- **🛑 TS-ошибки в `result.tsx` + 2 других** (Apr 16 timeline) — после рефакторинга `services/api.ts` экраны вызывают удалённые функции
- **GDPR 410 delete/restore** — нет UI ни в settings, ни вообще
- **Stripe в приложении** — текущий `upgrade.tsx` просто открывает `cleanway.ai/pricing?plan=X` в браузере. App Store может не пропустить (правило 3.1.1 — нельзя outside payments). Решение: либо StoreKit 2 + Google Play Billing (через RevenueCat), либо честное "buy on web" с правильной формулировкой
- **Push notifications** — `expo-notifications` подключён в `app.json`, но нет handlers в коде, нет регистрации токена, нет роутинга алертов
- **Bloom filter on-device** — заявлен, но не реализован. Сейчас VPN дёргает API на каждый домен (батарея!)
- **0 тестов на app-layer** (только native DNS)
- **Result screen** — упоминается timeline-стуб, нужно проверить и нормально вывести threat details
- **App Store listing** — `STORE_LISTING.md` есть, но не отправляли

## 5. EMAIL PLUGIN (Outlook Task Pane)

### Готово (где смотреть)
- Source: `email-plugin-outlook/`, deployed на `landing/public/outlook/` через `scripts/sync-addin.cjs` (prebuild hook)
- Прод-домен: `https://addin.cleanway.ai/outlook/`
- Manifest v1.1 (broad compat), permission `ReadItem` only
- `taskpane.js` — открывает sidebar, читает `Office.context.mailbox.item`, POSTит на `/api/v1/email/analyze`, рендерит verdict
- `commands.js` — ribbon button "Report phishing" → fire-and-forget на `/api/v1/feedback/report`
- Webmail banners (gmail.com, outlook.live.com/office.com, mail.yahoo.com) — через `extension-core/src/content/webmail.js`, MutationObserver
- Backend: `/api/v1/email/analyze` принимает headers+body, длина-капы, IP-rate-limit, circuit breaker на Safe Browsing
- Backend тесты: `test_email_{analyzer,rate_limit,i18n,validator,render_parity,unsubscribe}.py`

### Что осталось
- **🛑 Иконки** — `email-plugin-outlook/assets/` пустая. AppSource не примет без `icon-{16,32,80,128}.png`. **Sideload работает без них**, только в превью будет broken-image. Это первое что нужно сделать.
- **🛑 Захардкоженный английский** — 7 строк в `taskpane.js:216-232` ("This email looks safe", "Suspicious...", "Likely phishing", "Rescan", etc). Бэкенд-шаблоны email на 10 локалях, а сам плагин — нет.
- **Нет staging-манифеста** — есть только прод `addin.cleanway.ai`. Любая правка идёт сразу в продакшен.
- **Нет SSO** — `Office.auth.getAccessToken()` не подключён. Outlook-пользователи проходят как anon → IP-rate-limited, без daily quota.
- **0 frontend-тестов** — `taskpane.js` / `commands.js` / `webmail.js` без тестов вообще.
- **CSP headers** — не настроены в `next.config.ts`. AppSource может зарубить ревью.
- **Mobile Outlook** — манифест поддерживает, но не тестировали. Office.js на мобайле сильно урезан.
- **Attachment scanning / folder classification / per-org rules** — в роадмапе, не сделано (требуют `ReadWriteItem` permission и AppSource re-review)

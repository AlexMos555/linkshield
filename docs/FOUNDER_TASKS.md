## 🟠 №2 — Одна галка в настройках репо
**Время:** 30 секунд. **Разблокирует:** 1 крон (низкий приоритет — см. поправку).

### Зачем
GitHub Actions сейчас **не имеет права создавать PR** → падает крон
`refresh-brand-favicons` (обновляет хеши фавиконок для детекта клонов брендов,
Strategy #2). Сам скрипт я уже починил (падал 4 недели из-за импорта api-цепочки в
httpx-only крон) — осталась только эта галка.

> **Поправка (был неправ):** я сначала написал, что эта же галка блокирует
> `weekly-benchmark` (публикация честного recall-номера). **Это неверно** — бенчмарк
> коммитит напрямую (`git push`), и он **работает**: только что залил свежий бенчмарк,
> а ML переобучился до **AUC 0.9895**. Так что настройка нужна только фавиконкам →
> приоритет низкий, не срочно.

### Как
GitHub → репо `linkshield` → **Settings → Actions → General → Workflow permissions** →
галка **«Allow GitHub Actions to create and approve pull requests»** → Save.

## 🟡 №4 — Ответить: Apple Developer аккаунт — Organization или Individual?
**Время:** 1 мин (посетить developer.apple.com → Membership). **Разблокирует:** всю iOS-стратегию.

### Зачем
App Review **правило 5.4**: *«VPN-приложения могут публиковать только разработчики,
зарегистрированные как организация»*. То есть:
- **Organization** → системный iOS DNS-VPN можно шипить (код уже написан).
- **Individual** → iOS-VPN **не пройдёт ревью никогда**, пока не переоформишь аккаунт
  (нужно юрлицо + D-U-N-S). Тогда iOS-стратегия = share-extension (уже готов) + SMS-фильтр.

⚠️ Важно: **для сборки и теста на СВОЁМ айфоне организация не нужна** — Individual
спокойно подписывает dev-сборку. Стоп-кран срабатывает только при публикации в App Store.

## 🔵 №5 — Собрать и протестить мобильные ветки на устройстве
**Время:** ~30–60 мин. **Разблокирует:** мобильную защиту (основной канал фишинга).

Две ветки готовы, обе прошли моё ревью (tsc + prebuild чисто; multi-agent adversarial
review поймал 7 багов в некомпилированном Kotlin — **включая compile-break** — все
починены). Осталась компиляция + живой девайс — этого я сделать не могу.

### 5a. `mobile-share-flow` — «Проверить в Cleanway» из шторки
**Зачем:** ноль стоп-кранов от сторов, работает на **обеих** платформах, и это
**единственный** путь на iOS достать ссылки, открытые внутри мессенджеров.

```bash
git checkout mobile-share-flow
cd mobile
nvm use 20                       # НЕ Node 25
rm -rf node_modules && npm install   # чинит monorepo-хоист, иначе prebuild падает
npx expo prebuild --clean
npx expo run:android             # и/или: npx expo run:ios --device
```
**Проверить:** шарнуть фишинг-ссылку из WhatsApp → Cleanway в шторке → открывается вердикт.
Чеклист в [`mobile/SHARE_FLOW.md`](../mobile/SHARE_FLOW.md). Работает → merge в main.

### 5b. `mobile-android-vpn` — системная защита «в момент открытия»
**Зачем:** это **то самое**, что ты хотел — ловит открытие ссылки в **любом** приложении,
включая встроенные браузеры WhatsApp/Telegram (все резолвят DNS через туннель).
**Нужен Android Studio / SDK** (Xcode тут не нужен).

```bash
git checkout mobile-android-vpn
cd mobile
nvm use 20
rm -rf node_modules && npm install
npx expo prebuild -p android --clean
npx expo run:android             # нужен ANDROID_HOME + девайс/эмулятор
```
**Проверить:** тапнуть щит на главной → системный диалог VPN-согласия → принять →
появится постоянная нотификация → открыть фишинг-ссылку **из WhatsApp** → не резолвится
(заблокировано), под щитом появится «Blocked <домен>». Полный чеклист в
[`mobile/MOBILE_VPN.md`](../mobile/MOBILE_VPN.md).

⚠️ **Kotlin я скомпилировать не мог** (нет Android SDK) — если вылезет ошибка компиляции,
скинь мне вывод, починю.

**После того как заработает** — для публикации в Play нужно: заполнить **VpnService
Declaration** в Play Console + демо-видео ≤90 сек + prominent in-app disclosure.
(Категория «device security» разрешена — это формальность, не гамбл.)

## ⚪ №6 — Решение: privacy-safe аналитика (да/нет)
**Время:** решение. **Разблокирует:** exit-разговор.

### Зачем
План называет это **«show-nothing блокером»**: acquirer в первую очередь смотрит на
MAU / retention / install-velocity. Сейчас у тебя **нет ни одного счётчика** — показать
нечего, даже если юзеры пойдут.

### Но
Это добавит анонимный «phone-home» в **privacy-first** продукт. Технически можно сделать
приватно (server-side агрегаты, HyperLogLog, без PII, без профилей) — но это **изменение
outward-поведения privacy-бренда**, поэтому решение твоё, я не стал делать сам.

Скажешь «да» → построю приватно и честно задокументирую.
